# SecureVideoAppECDH
## SecureStream: Hybrid Video Encryption System (Система гібридного шифрування відео)

**SecureStream** — це прототип програмного комплексу для захищеної передачі потокових мультимедійних даних. Проєкт розроблено на платформі **.NET 8** (Windows Forms) з використанням сучасної гібридної криптосистеми, що поєднує **ChaCha20-Poly1305** та **X25519**.

## 📋 Основні можливості

* **Гібридне шифрування**: Поєднання швидкості потокового шифру та надійності асиметричної криптографії.
* **Perfect Forward Secrecy**: Використання ефемерних (одноразових) ключів. Компрометація довгострокового ключа не дозволяє розшифрувати дані минулих сеансів.
* **Автентифікація даних (AEAD)**: Використання Poly1305 гарантує цілісність даних. Будь-яка спроба модифікації файлу (навіть на 1 байт) призводить до помилки розшифрування.
* **Візуалізація ентропії**: Унікальний модуль візуального аналізу якості шифрування в реальному часі (ефект "білого шуму").
* **Висока продуктивність**:
    * Використання асинхронного патерну **TAP** (Task-based Asynchronous Pattern) для збереження відгуку інтерфейсу.
    * Оптимізована робота з пам'яттю за допомогою `Span<byte>` для зменшення навантаження на GC.

## 🛠 Технологічний стек

* **Мова**: C#
* **Фреймворк**: .NET 8, Windows Forms.
* **Криптографія**: [Bouncy Castle](https://www.bouncycastle.org/) (реалізація алгоритмів).
* **IDE**: Visual Studio 2022.

## 🔐 Криптографічна архітектура

Система використовує наступні примітиви:

1.  **Узгодження ключів (Key Agreement)**:
    * Протокол: **ECDH** на кривій **Curve25519 (X25519)**.
    * Забезпечує 128-бітний рівень безпеки при 32-байтному ключі.
2.  **Шифрування (Encryption)**:
    * Алгоритм: **ChaCha20** (потоковий шифр).
    * Швидкість: 1.20–2.40 cpb (з інструкціями AVX2).
3.  **Формування ключа (KDF)**:
    * Алгоритм: **SHA-256** для хешування спільного секрету.

### Структура захищеного контейнера (.enc)

Кожен зашифрований файл має заголовок для автономного розшифрування отримувачем:

| Поле | Розмір | Опис |
| :--- | :--- | :--- |
| **Length** | 4 байти | Довжина публічного ключа |
| **Ephemeral PubKey** | 32 байти | Одноразовий відкритий ключ відправника (X25519) |
| **Nonce** | 12 байт | Унікальна сіль для захисту від атак повторного відтворення |
| **Payload + Tag** | Змінний | Зашифровані дані + 16 байт тегу Poly1305 |

## 🚀 Встановлення та запуск

1.  **Клонуйте репозиторій**:
    ```bash
    git clone https://github.com/FullyRealist/SecureVideoAppECDH.git
    ```
2.  **Відкрийте проєкт**:
    Запустіть файл рішення `.sln` у Visual Studio 2022.
3.  **Залежності**:
    Переконайтеся, що пакет `BouncyCastle.Cryptography` встановлено через NuGet.
4.  **Запуск**:
    Скомпілюйте та запустіть проєкт (F5). При першому запуску програма автоматично згенерує пару ключів користувача у файл `ecdh.keys`.

## 🖥 Використання

1.  Натисніть **"Обрати файл"** та виберіть відеофайл.
2.  Натисніть **"Зашифрувати"**.
    > Програма згенерує унікальний ефемерний ключ.
    > У вікні візуалізації з'явиться "білий шум", що демонструє високу ентропію шифротексту.
3.  Для перегляду виберіть файл `.enc` та натисніть **"Розшифрувати"**.
    > Відео відтвориться лише якщо перевірка цілісності Poly1305 пройде успішно.

## ⚠️ Відмова від відповідальності

Цей проєкт є науково-дослідним прототипом. Хоча використані алгоритми є стандартами індустрії (RFC 7539, RFC 7748), програмна реалізація не проходила професійний аудит безпеки.
