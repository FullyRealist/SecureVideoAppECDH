# Анализ кода и предложения по улучшению

Ниже — краткий технический разбор текущей реализации SecureVideoAppECDH и практические шаги для улучшения надежности, безопасности и UX.

## Что уже хорошо

- Используется современная гибридная схема: X25519 + ChaCha20-Poly1305.
- Есть проверка целостности через AEAD (Poly1305 tag).
- UI не блокируется во время криптоопераций (`Task.Run` + `Progress`).
- Добавлена наглядная визуализация энтропии, полезная для демонстрационных целей.

## Наблюдения и риски

1. **Шифрование/дешифрование читает весь файл в память** (`File.ReadAllBytes`).
   - Риск: высокое потребление RAM на больших видеофайлах и вероятные OOM/GC-паузы.

2. **Минимальная валидация заголовка контейнера при дешифровании**.
   - Риск: поврежденные/поддельные файлы могут приводить к неочевидным ошибкам.

3. **Приватный ключ хранится в обычном файле рядом с приложением**.
   - Риск: компрометация ключа при доступе к файловой системе.

4. **Определение зашифрованного файла по расширению чувствительно к регистру**.
   - Риск: непредсказуемое поведение для `.ENC`, `.Enc` и т.д.

5. **Локализация и кодировка строк в исходниках** частично повреждены (в комментариях/логах есть “кракозябры”).
   - Риск: ухудшение сопровождаемости и UX.

## Что изменено в этом коммите

- Усилена валидация формата контейнера при дешифровании:
  - Проверка длины публичного ключа в заголовке.
  - Проверка факта полного чтения public key и nonce.
  - Проверка минимального размера payload (наличие AEAD-тега).
- Исправлена логика определения `.enc` с `OrdinalIgnoreCase`.
- Упрощено формирование имени выходного файла при расшифровке через `Path.ChangeExtension`.
- Поле выбранного файла сделано nullable, чтобы лучше соответствовать `Nullable`-режиму проекта.

## Рекомендуемый следующий этап (безопасность + производительность)

1. **Перейти на потоковую обработку файлов** (chunked encryption/decryption).
2. **Ввести версионированный формат контейнера**:
   - magic bytes,
   - версия,
   - алгоритм/параметры,
   - возможно, AAD для защиты метаданных.
3. **Защитить ключи пользователя**:
   - Windows DPAPI (`ProtectedData`) или Windows Certificate Store.
4. **Добавить тесты**:
   - успешный round-trip,
   - детект изменения 1 байта,
   - обработка битых заголовков,
   - большие файлы (stress).
5. **Привести проект к единой кодировке UTF-8** и унифицировать язык UI/README.
